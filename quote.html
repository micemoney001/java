    accepted convention in JavaScript and many other programming languages,
    making your code more familiar to other developers.    		 Simplicity: Single quotes are simpler and require
    less effort to type compared to double quotes.    		 Template literals: Backtick quotes allow you to create
    template literals, which make it easier to include
    variables and expressions within a string.
    2 HTML compatibility: When working with HTML attributes that require
    double quotes, like href or src, using double quotes in JavaScript strings
    can make it easier to integrate your code with HTML.     		Character clarity: Single quotes make it clearer to
    differentiate between quotes used within a string and
    the string itself. For example, if you want to include a
    quote within a string, you can easily use single
    quotes without the need for escaping.     		String interpolation: With backtick quotes,
    you can directly embed variables and expressions
    within a string using ${}, making
    string concatenation more concise.
    3 JSON consistency: JSON, which is a popular data interchange format,
    requires double quotes for key-value pairs, so using double quotes
    in JavaScript makes it consistent when working with JSON.     		Compatibility with HTML:
    Single quotes are widely used in HTML attribute
    values, so using them in JavaScript strings can
    make integration with HTML code smoother.    		 Multiline strings: Backtick quotes enable
    you to create multiline strings without the need
    for escaping line breaks, making the code
    more readable.
    3 String interpolation: Double quotes allow for easy
    string interpolation by concatenating variables or
    expressions within the string using the + operator.     		Consistency with other programming languages:
    Many programming languages,
    such as Python and Ruby, commonly use single quotes
    for string literals. By using single quotes in JavaScript, you
    can maintain a consistent coding style across different
    languages.     		Preserve whitespace: Backtick quotes
    preserve leading spaces, making it easier
    to format and align text within
    the string.
    3 Escaping single quotes: If your string contains single quotes,
    using double quotes can eliminate the need for escaping them,
    making the code cleaner and more readable.     		Easier string concatenation: When concatenating strings,
    using single quotes can be more convenient, as you can use double
    quotes within the string without escaping them.    		 Embedding special characters: Backtick quotes
    allow you to easily include special characters, such as
    newlines (\n) and tabs (\t), within a string.
    3 Widely understood: Double quotes are familiar to developers
    and easier to understand, especially for beginners who may not be
    familiar with the differences between single quotes and backticks.    		 Familiarity: Single quotes are
    a popular choice among developers,
    so using them can make your code more
    familiar and easier to understand for others.     		Enhance readability: Backtick quotes
    can improve the readability of your code by making
    it clear which parts are static strings and which
    parts are placeholders or expressions.
    3 Compatibility with external libraries: Some external libraries
    or frameworks may assume double quotes for string parameters or
    configurations. Using double quotes ensures compatibility with such
    libraries.    		 Code readability: Some developers argue
